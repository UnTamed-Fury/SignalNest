# Code Audit Report: SignalNest (mr_notifier)
**Date:** 2026-02-22
**Auditor:** Gemini CLI
**Overall Score:** 35/100 (Critical Improvements Needed)

---

## Executive Summary
The SignalNest project is currently in an early development phase, despite the roadmap claiming Phase 1 completion. While the project uses a modern tech stack (Hono, Supabase, Jetpack Compose), it suffers from severe implementation gaps, critical logic bugs, and a complete absence of testing. A significant portion of the server-side architecture (DSL engine, adapters, delivery modules) exists only as empty directory structures.

---

## Critical Issues (Must Fix Immediately)

1.  **Production Runtime Crash:** In `notificationRoutes.get('/')`, the variables `start` and `end` are defined only within the `isMock()` block. In production (Supabase) mode, these variables are out of scope, causing a `ReferenceError` and immediate crash on every notification list request.
2.  **State Loss:** The Android application (`forked-version`) uses `MutableStateFlow` in the repository without a persistence layer (Room). All user data, including notes and todos, is lost when the app process is killed.
3.  **Sync Logic Bug:** In `sync.routes.ts`, the "Real Supabase mode" incorrectly reads `currentVersion` from the in-memory `mockSyncState`. This causes the sync version to reset every time the server restarts, leading to potential data duplication or loss on clients.
4.  **Zero Test Coverage:** No unit, integration, or E2E tests exist for either the server or the Android application. This makes the project extremely brittle and risky to refactor.

---

## Major Issues

1.  **"Vaporware" Features:** Many directories in the server (`dsl/`, `adapters/`, `modules/sources/`, etc.) are empty. The roadmap marks Phase 1 as COMPLETED, yet core promised features like device registration and GitHub webhooks are placeholders.
2.  **Hardcoded Credentials/IDs:** The Android app hardcodes `userId = 0` in repository calls, bypassing the authentication system.
3.  **Data Type Inconsistency:** The server returns `createdAt` as a timestamp (number) in mock mode and an ISO string in Supabase mode. This will cause parsing failures in the Android client depending on the server environment.
4.  **Security Gaps:** 
    - `isMinifyEnabled` is disabled in the Android release build.
    - JWT verification is duplicated across routes instead of using a centralized middleware.
    - `bcrypt` salt rounds (10) are below the modern recommended standard (12+).

---

## Minor Issues & Technical Debt

1.  **Dead Scripts:** `package.json` contains scripts (`db:init`, `db:migrate`) that point to non-existent files.
2.  **Redundant Dependencies:** `build.gradle.kts` lists `datastore-preferences` twice.
3.  **Basic Sync Implementation:** The sync UPSERT logic on the Android client is $O(N^2)$ in memory, which will degrade performance as the notification count grows.
4.  **Incomplete FCM Integration:** `SignalNestMessagingService.onNewToken` is a TODO, meaning the server cannot reliably send push notifications to specific devices yet.

---

## Long-Term Risks

- **Architectural Divergence:** The "Mock Mode" and "Real Mode" logic paths are heavily branched. As the project grows, maintaining parity between these two will become impossible, leading to "works in dev, fails in prod" scenarios.
- **Scalability:** The current sync model (full list merge) will not scale beyond a few hundred notifications.
- **Maintenance Cost:** Without tests, the cost of adding features will grow exponentially as side effects become harder to track.

---

## Suggested Roadmap Updates

1.  **Integrate Room DB:** Priority #1 for Android to ensure data persistence.
2.  **Fix Production Scoping Bugs:** Ensure the server can actually run with Supabase.
3.  **Implement Test Suite:** Add Vitest for the server and JUnit/Compose Test for Android.
4.  **Unify Data Access:** Use a Repository pattern on the server to hide the mock/Supabase distinction from the route handlers.
5.  **Proper Auth Flow:** Link the Android login UI to the repository and stop using `userId = 0`.
